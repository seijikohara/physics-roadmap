## Claude Code 固有の指針

- 解説の章を書くときは`chapter-writer`サブエージェントを使う。`chapter-writer`が 4 つの検証（数式`math-verifier`・論理飛躍`logic-leap-verifier`・視覚表現`visual-verifier`・書式`markdown-verifier`）を呼ぶ。
- 章の執筆・加筆・改稿・監査では、AGENTS.md の自己完結ルールを徹底する。とくに次を必ず守る。
  - **参照は上流のみ（前方参照の禁止）**: 各章は推移的に依存する上流の章だけを参照・言及する。下流の章・未習概念・後続カテゴリ（未執筆の物理カテゴリを含む）へ言及しない。章末に「次の章では〜」式のリンク・予告を置かない。
  - **前提の網羅**: 本文で使う上流の概念・記号（集合・絶対値・関数・象限など）を導入した上流章を、前提節に漏れなく挙げる。
  - **記号の初出**: 記号は初出で読み方・意味・（正式記法か便宜的書き方かの）区別を示す。
  - **内部IDの非露出**: 本文・図・脚注に`M-SET`・`M-SEQ`などのカテゴリ識別子を書かない。
  - **図ラベル**: 数式でない日本語を`\text{}`で囲まない（素の日本語で書く）。
- 成果物・コミットメッセージ・PR・チャット応答は日本語で書く。
- 複数コミットにわたる開発は、メインの作業ツリーを汚さないよう worktree で隔離する。
