# 可視化スタック設計

物理学学習ロードマップ（Astro + Starlight、静的サイト生成、MDX、React、KaTeX）の図解に用いる可視化ライブラリの選定と、chapter-writer が使うコンポーネント方針を定める。

## 改訂

- 2026-06-24: 2D 標準を JSXGraph から D3 へ改めた。サイトは GitHub Pages の静的サイトであり、ベン図など教示用の図ではドラッグの利得が希薄なため、ビルド時に静的 SVG を焼き込む方式を既定とする。JSXGraph と `jsxgraph` 依存は撤去した。

## 方針

- **ビルド時静的 SVG を既定**とする。図はビルド時に SVG を生成して HTML へ焼き込み、クライアント JS を載せない。静的サイト（GitHub Pages）で読み込みが速く、アクセシビリティと堅牢性が高い。
- **2D は D3 に統一**する。関数・幾何・ベクトル・ベン図・数直線・データグラフを、D3 のモジュール（`d3-scale`・`d3-shape`・`d3-path`など）で計算し、静的 SVG として出力する。図ごとの追加依存（venn.js など）は使わない。
- **双方向性は例外**とする。ドラッグやスライダーが本当に要る図のみ、`d3-selection`・`d3-drag`でクライアント側をハイドレートする。
- **宣言的コンポーネント中心**とする。再利用可能な React コンポーネントを用意し、chapter-writer は MDX から JSON シリアライズ可能な props で呼ぶ。
- **3D は別枠**とする。曲面・3D ベクトル場・時空など真の 3D は、物理段階で react-three-fiber を導入する。

## 層とライブラリ

| 層                                                    | 採用                                               | 導入時期               |
| ----------------------------------------------------- | -------------------------------------------------- | ---------------------- |
| 2D 数学・幾何（関数・幾何・ベクトル・ベン図・数直線） | D3（ビルド時静的 SVG。自作の宣言的コンポーネント） | 今                     |
| データグラフ                                          | D3（ビルド時静的 SVG）                             | データ図が必要な章から |
| 概念図・フロー                                        | Mermaid（現用）                                    | 継続                   |
| 3D（曲面・3D ベクトル場・時空）                       | react-three-fiber + drei                           | 物理段階               |
| 数式                                                  | KaTeX（現用）                                      | 継続                   |

## 選定根拠（2026 時点）

- D3: 2D 描画とデータの汎用ツールキットで、ベン図・関数・幾何・ベクトル場・位相図・データグラフを 1 つの基盤で表現できる。モジュールをビルド時の計算に使えば、出力は静的 SVG となりクライアント JS を載せない。活発に保守され、エコシステムが大きい。低レベルな API は高レベルコンポーネントで隠蔽する。
- データグラフ: 静的なデータ図は D3 の静的 SVG で描く。ツールチップなど双方向性が要るデータ図が必要になった時点で、描画手段を選定する。
- react-three-fiber: JSX が Three.js に 1 対 1 で対応する宣言的レンダラで、React 19 に対応し活発に保守される。真の 3D のみで導入する。
- Mermaid: LLM 量産性が最良で活発に保守される。概念図は継続して使う。
- 不採用: JSXGraph・Mafs（2D を D3 に統一）、venn.js・chartjs-chart-venn（面積比例のデータ駆動図で教示用途に不向き）、Chart.js・Plotly・ECharts（データグラフは D3 の静的 SVG で賄う。双方向性が必要になった時点で選定する）。

## コンポーネント設計

- 各図は **意味的な高レベルコンポーネント**として提供する（例: `VennDiagram`・`FunctionGraph`・`NumberLine`・`VectorDiagram`・`DataChart`）。
- D3 系コンポーネントはビルド時に静的 SVG を生成する。`d3-scale`・`d3-shape`・`d3-path`で座標とパスを計算し、React で SVG 要素を返す。`client:only`は付けない。
- **双方向性**: ドラッグやスライダーが要る図のみ、クライアントアイランドとして `d3-selection`・`d3-drag`でハイドレートする。
- **テーマ追従**: Starlight の light/dark に合わせ、SVG のラベルを CSS で `currentColor` 基調に整える。ラベルのフォントは KaTeX に合わせる。
- **アクセシビリティ**: 各 SVG に `aria-label` を付ける。
- 各コンポーネントは「1 つの明確な役割・明示的な props インターフェース・独立して理解可能」を満たす単位に分割する。

## 移行

- `VennDiagram` を静的 SVG コンポーネントとして実装済み（M-SET-01 で利用）。JSXGraph と `jsxgraph` 依存、旧 Mafs デモコンポーネントは撤去済み。Chart.js ベースの `DataChart` と `chart.js`・`react-chartjs-2` 依存も、未使用のため撤去した（YAGNI。データ図が必要になった時点で再導入する）。
- D3 のモジュールを順次導入し、`FunctionGraph`・`NumberLine`・`VectorDiagram`・`DataChart`を整備する。
- データグラフは D3 の静的 SVG で実装する。
- react-three-fiber の依存追加と利用は、3D が必要な物理段階まで保留する。
