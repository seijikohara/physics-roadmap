# 学習ロードマップ設計書

本書は物理学ロードマップの全体設計を定める。章執筆の唯一の正とする。章は本書の依存順に従って書く。

## ベースラインとゴール

- ベースライン: 本ロードマップが前提とする知識。特定の学年や課程に依らず、カテゴリ「前提とする知識」（PRE-01）で算数・数学・理科の前提を具体的に明記する。
- ゴール: 次の2つを数式として理解する。
  - 標準模型のラグランジアン。各項の意味と導出の筋を追える水準に至る。
  - 一般相対性理論。アインシュタイン方程式を読み、シュワルツシルト解・重力波・宇宙論の初歩まで追える水準に至る。

## 設計原則

### 自己完結

- 一連の章のみでゴールへ到達できる構成とする。
- 説明の論理を外部資料に委ねない。「詳細は専門書を参照」と書かない。
- 前提とする知識に含まれない数学（微積分・三角関数・ベクトルなど）も、使う前に章内で導入する。
- 完全な厳密証明が過大になる箇所は、直観と結果の承認で扱い、射程外であると明示する。
- 実験で決まる量（結合定数・粒子の質量など）は導出しない。所与の入力値として提示し、理論の論理を途切れさせない。

### 演習による習得

- 到達水準は「演習で身に付く」とする。読んで終わりにしない。
- 各章に例題と演習問題を置く。演習には解答を付ける。

### 一次資料への脚注

- さらに学ぶための一次資料・データ出典を脚注で示す。
- 脚注は信頼できる一次資料に限る。原論文・標準的な教科書・公的機関のデータなどを対象とする。
- 脚注は補足とする。論理の鎖の一部にはしない。脚注を読まなくても本文だけで理解できる構成を保つ。

## モジュール体系

- モジュールは36カテゴリに分ける。各カテゴリは単一の分野とする。前提トラックは1、数学トラックは23、物理トラックは12である。
- ID は「トラック接頭辞-分野コード-連番」で表す。数学トラックは M、物理トラックは P を接頭辞に付ける（例: M-LIN-04、P-QM-03）。各モジュールはおおむね1レッスン（教科書の小節相当）とし、1章に対応する。
- 前提列は原則として ID で記す。カテゴリをまたぐ参照も ID を用いる。導入元を単一モジュールに特定しない題材は名称で記す（例: 三角関数、特殊相対論の4元運動量）。前提が前提知識のみのモジュールは「前提知識」と記す。
- 総モジュール数は410とする。

## カテゴリの依存と推奨順序

各モジュールの前提が正確な依存を表す。大まかな履修順は次のとおりとする。

## クリティカルパス

ゴールは2つある。標準模型（P-SMD-14）と一般相対性理論（P-GR-22）である。共通基盤は数学トラック前半に置く。確率統計（PRB）はどちらにも必須ではない。

### 標準模型（P-SMD-14）への最小経路

> M-SET → M-ALG → M-FUN → M-CALC → M-MVC → M-LIN → M-ODE → M-CXA → M-SPF → M-VAR → M-GRP → M-REP → M-LIE → M-DG → P-CM → P-AM → P-EM → P-SR → P-QM → P-RQM → P-QFT → P-GAU → P-SMD

P-SMD-14 では3本の柱が合流する。

1. ゲージ原理: P-EM-22 → P-QFT-15 → P-GAU-02 → P-GAU-05 → P-SMD-02・P-SMD-05
2. 物質場: P-RQM-04 → P-QFT-13 → P-SMD-01 → P-SMD-09
3. 対称性の破れ: P-GAU-09 → P-GAU-10 → P-GAU-11 → P-SMD-03

### 一般相対性理論（P-GR-22）への最小経路

> M-SET → M-ALG → M-FUN → M-CALC → M-MVC → M-LIN → M-ODE → M-PDE → M-VAR → M-DG → M-RG → P-CM → P-EM → P-SR → P-GR

P-GR-22 は量子論（P-QM・P-RQM・P-QFT・P-GAU・P-SMD）を必要としない。一般相対性理論は標準模型から独立した枝である。

## 章テンプレート

各章は次の構成とする。

1. frontmatter: `title`は必須、`description`は任意とする。
2. 前提: 依存モジュールへのリンクを冒頭に置く。
3. 学習目標: 章で読めるようになる数式・概念を箇条書きで示す。
4. 本文: 導入から展開へ進める。表現手段は次のとおり使い分ける。
   - 数式: KaTeX（インライン`$...$`、ブロック`$$ ... $$`）。
   - 概念図・フロー: Mermaid のコードブロック。
   - 2D 数学・幾何（関数・幾何・ベクトル・ベン図・数直線）: ビルド時に静的 SVG を生成する React コンポーネント（`VennDiagram`など）。標準ライブラリは D3 とし、`client:only`は付けない。
   - データグラフ: D3 によるビルド時の静的 SVG を標準とする。データ図が必要な章で導入する。
   - 3D（曲面・3D ベクトル場・時空）: 物理段階で react-three-fiber を導入する。
   - 表現手段の選定方針は[可視化スタック設計](superpowers/specs/2026-06-24-visualization-stack-design.md)に従う。
5. 例題: 解法を示す。
6. 演習問題: 解答を付ける。
7. まとめ: 後続モジュールへのリンクを置く。
8. 脚注: 一次資料・データ出典を示す。
